## Core idea

A mapping is a rule that relates elements from a domain to a range. For a mapping to be defined strictly as a **function**, each input in the domain must map to exactly one unique output in the range. If one input maps to multiple outputs (one-to-many), it is **not** a function.

## Key formulas

$$\text{Domain: } X, \quad \text{Codomain: } Y$$

$$f: x \mapsto y \quad \text{or} \quad f(x) = y \quad \text{where } x \in X$$

## Mapping types

**One-to-one** · each input maps to a unique output. Passes both vertical and horizontal line tests.

**Many-to-one** · several inputs share the same output. Still a valid function — passes the vertical line test.

**One-to-many** · one input maps to multiple outputs. **Not a function** — fails the vertical line test.

## Steps / method

Identify the given domain (the set of valid inputs) and range (the target set).

Apply the **vertical line test** on the graph: draw a vertical line anywhere across the domain.

If any vertical line intersects the graph at more than one point, classify it as one-to-many and conclude it is **not a function**.

If it passes the vertical line test, apply the **horizontal line test** to determine whether it is one-to-one or many-to-one.

### Key rule

A mapping is a function only if every input has exactly one output. Always state the domain when defining a function.

## Worked example

Does the mapping $f(x) = \pm\sqrt{x}$ define a function on $x \ge 0$?

For $x = 4$, the outputs are $f(4) = 2$ and $f(4) = -2$ — two outputs for one input.

This is a **one-to-many** mapping, so it is **not a function**.

## Examiner tip

Examiners frequently test whether a function is valid over a specific domain. A mapping can be a function under one restricted domain but invalid over an unrestricted one — e.g. $f(x) = \sqrt{x}$ is only a function for $x \ge 0$.

Do not confuse $f^{-1}(x)$ (the inverse function) with $[f(x)]^{-1}$ (the reciprocal). These are entirely different and a common source of lost marks.

## Quick check

If a horizontal line crosses a function's graph more than once, then the function is classified as a **many-to-one** function.
